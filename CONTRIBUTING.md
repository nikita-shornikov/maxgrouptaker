# Руководство по участию в проекте (Contributing)

Спасибо за интерес к **MAX Group Taker** (добавление в группу MAX по имени через веб). Ниже — как внести вклад в проект.

## Как участвовать

1. **Сообщения об ошибках** — создавайте Issue с описанием шагов воспроизведения и окружения (ОС, Python, версия Playwright).
2. **Предложения** — идеи по улучшению скрипта или документации приветствуются в Issues.
3. **Код** — через Pull Request (см. ниже).

## Разработка

### Окружение

- Python 3.10 или новее.
- Клонируйте репозиторий и установите зависимости:

```bash
git clone <url репозитория>
cd maxgrouptaker
pip install -e ".[dev]"
playwright install chromium
```

### Запуск

Из корня репозитория:

```bash
python script2_web_add_to_group.py --chat-id -123456789 --name "Тест"
```

### Стиль кода

- Код в стиле PEP 8, длина строк до 100 символов (см. `pyproject.toml` → `ruff`).
- Проверка: `ruff check src/ script2_web_add_to_group.py` и при необходимости `ruff format src/ script2_web_add_to_group.py`.

### Тесты

- При добавлении логики желательно добавить тесты (например, `pytest`).
- Запуск: `pytest` из корня проекта.

### Pull Request

1. Создайте ветку от `main` (или текущей базовой ветки).
2. Внесите изменения, по возможности добавьте или обновите тесты и документацию.
3. Убедитесь, что код проходит линтер и тесты.
4. Опишите в PR, что изменено и зачем.
5. После ревью изменения могут быть смержены в основную ветку.

## Конфигурация и секреты

- Не коммитьте `config.yaml` с персональными данными (он уже в `.gitignore`).
- В PR не должно быть токенов, паролей и персональных данных.

## Вопросы

Если что-то непонятно — создайте Issue с меткой «question» или «documentation».
